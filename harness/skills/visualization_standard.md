# 输出格式规范 — 图表 + PDF 报告 + Dashboard

> 🔵 ChatBI 平台标准。所有可视化输出和报告生成均需遵守。
> 适用技能：`document-pdf`、`elegant-reports`、`build-dashboard`、`humanizer-zh`

---

## 一、图表规范

### 配色
- 主色：`#1a5276`（深蓝）/ `#2e86c1`（中蓝）/ `#e67e22`（橙）
- 分类色板：蓝 → 橙 → 绿 → 紫 → 红（依次使用，不重复色系）
- 正/负值：绿色（`#27ae60`）表示正向，红色（`#e74c3c`）表示负向
- 中性值：灰色（`#95a5a6`）用于不显著/无变化的数据

### 图表要素（每张图必须包含）
- **标题**：14pt 加粗，描述"什么数据 + 什么维度 + 什么时间"
- **轴标签**：11pt，含单位（如"Units Sold"、"Price ($)"）
- **图例**：8-9pt，放置在图表右侧或下方，不与数据区域重叠
- **数据来源**：7pt 灰色，图表右下角标注数据来源文件

### 导出
- 分辨率：DPI ≥ 150
- 格式：PNG（用于嵌入报告）、可保留 SVG 源文件
- 文件名：`<序号>_<描述>_<维度>.png`，如 `01_weekly_sales_by_category.png`

## 二、PDF 报告格式规范

### 页面设置
- 纸张：A4（210 × 297 mm）
- 页边距：上下 18mm，左右 20mm
- 字体：微软雅黑（中文）/ Helvetica（英文数字），正文 9.5pt，行距 1.6 倍
- 页眉：7pt 灰色，左侧文档标题，右侧不显示
- 页脚：7pt 灰色，居中"第 X 页"

### 间距规则（消除大量空白的关键）
- **标题后**：标题下方 6-8mm 间距，不额外加空行
- **段落间**：段间距 4mm，不使用多个换行
- **表格前/后**：表格上下各 4mm 间距
- **图片前/后**：图片上下各 6mm 间距
- **章节间**：新章节前一个 PageBreak，不断在页面中间留白
- **避免半页空白**：如果当前页剩余空间 < 40%，将大表格或图片推到下一页

### 表格规范
- 表头：深蓝底（`#1a5276`）白字，9pt 加粗
- 数据行：交替白/浅蓝（`#f4f8fb`）背景，8-9pt
- 数字列：右对齐，保留合理小数位（百分比 1 位，金额 2 位，整数 0 位）
- 表格宽度：不超过页面宽度的 90%
- 每张表下方标注"表X：说明"（7pt 灰色斜体）

### 图片嵌入
- 图片宽度：页面宽度的 80-100%（大图 100%，小图并排 48%）
- 图片高度：不超过页面高度的 50%
- 嵌入方式：base64 内嵌或相对路径引用，确保 PDF 可独立分发

### 内容层级
```
封面 → 目录 → 执行摘要(1页) → 
  第1章 EDA → 第2章 预测 → 第3章 ABC/XYZ →
  第4章 安全库存 → 第5章 促销 → 第6章 竞争定价 →
  策略建议 → 附录（框架说明/数据来源/假设声明）
```

### 文字规范（配合 humanizer-zh）
- 禁用模糊定性词："显著提升""明显下降""很好的表现"→ 必须配具体数字
- 禁用夸张修饰："极其重要""关键里程碑""不可或缺"
- 禁用协作痕迹："希望这对您有帮助""请告诉我""当然！"
- 数字格式：千分位逗号（73,100）、百分比保留 1 位（6.1%）

## 三、Dashboard 格式规范

### 布局
- KPI 卡片：顶部一行 4-5 个，等宽排列
- 图表区：下方 2-3 列网格，每个图表带标题
- 明细表：最下方全宽展开
- 筛选器：左侧栏或顶部栏

### 交互
- 筛选器联动：选择品类后，所有图表同步过滤
- 钻取支持：品类 → 产品 → SKU
- Hover 提示：数据点悬停显示具体数值

### HTML 输出
- 自包含：图片内嵌为 base64，零外部依赖
- 响应式：≥ 3 个断点（900px 以下折叠为单列）
- 色系与 PDF 保持一致

## 四、输出前检查清单

- [ ] 所有图表有标题、轴标签、图例、数据来源
- [ ] PDF 无大段空白（页面利用率 > 60%）
- [ ] 表格表头深蓝底白字，行交替色
- [ ] 数字格式统一（千分位、小数位）
- [ ] 文字无 AI 痕迹（已通过 humanizer-zh 检查）
- [ ] 图表配色与规范一致
- [ ] 报告含数据来源和假设声明
- [ ] 文件名规范，版本号清晰

## 五、现有 PDF 报告格式问题及修复

> 2026-06-28 自检发现。记录于此确保后续输出遵守。

| 问题 | 表现 | 修复方式 |
|------|------|----------|
| JSON 字段不匹配 | 报告代码引用了 `baseline_avg`/`promo_avg` 但 JSON 中无此字段 → KeyError | 生成报告前先 `print(data.keys())` 确认字段名，代码中字段名与 JSON 严格一致 |
| 页面空白过多 | 章节间、表图间使用多个 Spacer 或换行 | 按本规范"间距规则"：标题后 6mm、段间 4mm、表前后 4mm、图前后 6mm。用 `PageBreak()` 而非空行换页 |
| 表格过宽溢出 | 部分表格列数过多导致超出页面 | 表格总宽度 ≤ 160mm（A4 宽度 210mm - 左右边距 40mm） |
| 图片过大占满页面 | 大图未控制高度 | `img_block()` 中 height=width*0.62，确保不超页面 50%
